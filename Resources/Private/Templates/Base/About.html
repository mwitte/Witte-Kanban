W:Kanban

This project stated from a personal idea to learn something about TYPO3 Flow. The main goal for this project is self-improvement. Features are not important, but the technical quality is. Domain-driven design and test-driven development are patterns which the development of TYPO3 Flow followed. This project follows the same techniques.

Features
  • Create a Kanban board
  • Create and delete (sub) columns
  • Create, edit, delete, archive Tickets
  • Switch tickets between columns
  • Responsive layout
  • Test coverage 100% for models
  • Test coverage over 90% for business logic
  • Recursive column structure(infinite nestable)
Planned features
  • Boards should be deletable
  • Multi-language support
  • Swim lanes
  • Drag and drop for tickets
  • File upload for tickets
  • Detailed statistics
  • TYPO3 Neos Plugin
